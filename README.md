## 野望

### 面白いプロダクトを自分自身が一番楽しんで開発できるエンジニア
いくつか会社を経験してきた中で、エンジニアをやっていて結局いちばん大事なのは作るものの面白さだと考えています。
特に面白いと感じるプロダクトは`新しい領域にアプローチしているプロダクト`、`影響度・影響範囲が広く、夢があるプロダクト`です。
自分自身も作っているものに対して魅力を感じて、自分のできることをすべてやりたいと思えるようなプロダクトを開発・指揮できるエンジニアになりたいです。

日々使われる技術は新しくなっていくため、自分が面白いと思うプロダクトの開発に携われるように技術のキャッチアップは欠かさないようにしていきたいです。

### 2021/1 - 現在 : 株式会社 AVILEN

職種: Web エンジニア

#### 自社 AutoML サービス構築
##### チームの規模
5、6 人(主な役割: 開発メンバー＋サブリーダー)
##### 主な技術
Python(3.8), React+Typescript, AWS(ECS,APIGW,Lambda,SQS,DynamoDB,RDS,S3,CF など),Terraform

##### 主に担当したこと
- オンプレに導入されているサービスの SaaS 化
- インフラ構成刷新
     - EC2+RDS構成からサーバーレスアーキテクチャに移行
     - 主な構成
       - API
         - APIGW+Lambda構成を採用
         - ローカルでの開発環境としてはFastAPIを採用し、Mangumというツールを利用することでAPIGW+Lambda構成とのインターフェースの差を吸収している
         - API認可にははcognitoを採用
       - ジョブキュー
         - APIから入稿されたジョブをSQSに入稿し、入稿をトリガーにLambda関数が発火し、裏側のECSスタンドアローンタスクが起動する用になっている
       - ジョブ実行
         - ECSのスタンドアローンタスクを利用
         - 当初はオートスケーリングで制御する予定だったが、タスクごとの実行時間や利用リソースにばらつきがあるため、スケールインなどで実行中のジョブが中断されるリスクを考慮し上記の構成にした。
       - 各種ストレージ
         - リレーショナル・非リレーショナルなエンティティによってRDS(AuroraDB)とDynamoDBを使い分けている
       - フロントエンド
         - ほぼ全てがログイン前提のページであるため、React+TSなSPAで実装
         - CF+S3構成で静的配信
- IaCによるインフラ管理
  - Terraformを使用して、基本的にすべてのインフラをコード化している
- チームに必要な人材確保業務
- スクラムに近い開発形式の整備、github projectsを用いたプロジェクト管理
- サービスの技術的仕様決め
  - 抽象的な要望に対する機能設計と成果物が要望にフィットしているかの確認
- チームが円滑に回るようにサポート
- 機能設計、機能開発
- CI/CDによる各種自動化

##### 成果
- サーバーレスなインフラ構築＋IaCでコストを抑えつつ、メンバーごとに開発環境を持てるようにした
- 各種定例の整備やリモート下でもメンバーが集まって話せるような場を提供することでチームの心理的安全性を向上させ、全員が自走しているチームにすることができた
